窥探JVM--Class文件结构

发表于 2016-05-16   |   分类于 java ， JVM   |     |   阅读次数

一、java跨平台特性

java语言成为当今之热门，其主要原因是它的跨平台特性，正如当初其宣传语所称“一次编译，到处运行”正是将java语言推上了热门榜，成为当今互联网开发的主力语言。那么为什么说java是具有跨平台的特性呢？我们知道对于不同的操作系统，不同的操作系统版本，其系统调用与底层结构与指令集都是大不相同的，所以程序的适配一直是一个大问题，而java使用了JVM作为运行环境，java文件经过编译后成为class文件，运行在JVM上，而只需要针对不同的系统适配不同的jvm就能够做到“一次编译，到处运行”，而jvm的适配，都由sun公司替我们解决，我们无需关心底层，只专注于业务。

二、class类文件结构

我们平时所编写的代码都是java文件，而JVM只能运行二进制字节码，所以java在编译时就会将.java文件编译成为.class文件，而class文件是运行在jvm中的，所以有着严格的格式要求，下面我们就来看一下class类文件结构.

class数据类型

Class文件经过编译后为一组二进制流，基于unicode进行编码，其基础单位是8位无符号字节码，所有数据都是按照java官方的class类格式紧凑的排列在class文件中，中间没有任何分隔符，对于超过基础单位8位的数据，将按照big-Endian的顺序储存（高位字节存储在低位的地址上），在class文件中存储的数据都是程序运行的必要数据，具体数据单位如下表所示：

使用该无符号单位可以用来描述具体数据、类的引用等，下面我们来看一下class类的具体结构。

class文件属性分析

magic

魔数，占用4个字节，其实在许多地方都会使用魔数，在这里表示这是一个class文件，用来校验使用。

minor_version

表示该class文件的次版本号，占用2个字节，与主版本号一起确定当前java的版本

major_version

主版本号，占用2个字节，与次版本号一起确定当前java的版本，从hex friend中可以看到，我的版本号为00000034，代表了JDK1.8.0

constant_pool_count

常量池中常量的数量，占用2个字节。由于常量池中的数据数量是不固定的，所以需要这个计数器，用来记录常量池中数据的数量，计数器从1开始计数，因为0表示该类不引用任何常量池数据。

constant_pool

常量池数据，其实就是一个数组，元素为常量数据的类型，这里的常量池与JVM中的常量池是不同的，Class文件中的常量池主要存储了一些数据的字面量（字符串，final常量或者某个属性的初始值）与符号引用（类和接口的全限定名，字段名等），从反编译Student实例中我们也可以看到，Constant pool里面存了各种类型的数据，主要类型有以下14类：

  • CONSTANT_Utf8_info tag标志位为1, UTF-8编码的字符串
  • CONSTANT_Integer_info tag标志位为3， 整形字面量
  • CONSTANT_Float_info tag标志位为4， 浮点型字面量
  • CONSTANT_Long_info tag标志位为5， 长整形字面量
  • CONSTANT_Double_info tag标志位为6， 双精度字面量
  • CONSTANT_Class_info tag标志位为7， 类或接口的符号引用
  • CONSTANT_String_info tag标志位为8，字符串类型的字面量
  • CONSTANT_Fieldref_info tag标志位为9, 字段的符号引用
  • CONSTANT_Methodref_info tag标志位为10，类中方法的符号引用
  • CONSTANT_InterfaceMethodref_info tag标志位为11, 接口中方法的符号引用
  • CONSTANT_NameAndType_info tag 标志位为12，字段和方法的名称以及类型的符号引用

access_flags

访问标志，占用2个字节，记录了该类或接口的访问信息，具体数值如下表所示：

标志名 值 含义
ACC_PUBLIC 0x0001 public类型
ACC_FINAL 0x0010 类为final类型
ACC_SUPER 0x0020 Invokespecial是否是使用动态绑定
ACC_INTERFACE 0x0200 接口类型
ACC_ABSTRACT 0x0400 抽象类类型
ACC_SYNTHETIC 0x1000 是否由用户代码产生
ACC_ANNOTATION 0x2000 表示是否为注解
ACC_ENUM 0x4000 表示为枚举

若有多个标签，那么是使用或运算来进行计算，因为我们可以看到访问标志的值为一个4位的十六进制数，而且分布在每一位的数据都是不同的，所以最终得到的值可能是一个复合状态。

this_class、super_class

类的索引与父类的索引，分别占2字节，指向常量池中的索引，在常量中的入口必须为CONSTANT_Class_info表，CONSTANT_Class_info的结构如下图所示

类型 数据名称 解释
u1 tag 标志位
u2 name_index 类的全限定名

interfaces_count

实现接口的个数，占用2个字节，该计数器包括该类自身实现个数与父类实现个数的总和

interfaces

实现接口的索引数组，其中的每一项都指向常量池中CONSTANT_Class_info结构

fields_count

字段描述符个数，占用2个字节，该计数器用于记录类变量和实例变量中字段的数量总和

fields

字段描述符数组，其中的每一项都指向field_info表的索引，只有在类与接口中声明的字段（类级变量或者实例级变量）才能被记录在fields数组中，该数组中不会列出超类或父类接口继承而来的字段。下面为field_info结构

类型 数据名称 解释
u2 access_flags 访问标示
u2 name_index 字段的全限定名，为指向常量池中的索引
u2 descriptor_index 字段描述符，为指向常量池中的索引
u2 attributes_count 字段总的属性表的数量
attribute_info attributes[attributes_count] 属性表

methods_count

类中声明方法的数量，占用2字节。

methods

方法描述符数组，其中每一项都指向method_info表的索引，可以看出method_info和field_info的结构是很类似的

类型 数据名称 解释
u2 access_flags 访问标示
u2 name_index 字段的全限定名，为指向常量池中的索引
u2 descriptor_index 字段描述符，为指向常量池中的索引
u2 attributes_count 字段总的属性表的数量
attribute_info attributes[attributes_count] 属性表

attributes_count

类中声明属性的数量，占用2字节。

attributes

属性描述符数组，其中每一项都指向attribute_info表的索引，它给出了在类或接口所定义的属性的基本信息具体结构如下表所示：

类型 数据名称 解释
u2 attribute_name_index 访问标示
u4 attribute_length 字段的全限定名，为指向常量池中的索引
u1 info 指向属性信息
