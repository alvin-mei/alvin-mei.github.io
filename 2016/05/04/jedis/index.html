Redis简介

什么是Redis

Redis是现在比较流行的Nosql之一，作为一个高速缓存系统，Redis使用key-value形式来存储数据。Redis支持的数据格式很多，能够储存String、Object、List、Set、Hash等数据类型，并且具有持久化功能，使用Redis能够提高我们系统对于常用不可变数据的读取效率，减轻关系型数据库的访问压力。

为什么要使用Redis

由于现在的系统面对的用户数量都是百万、千万级别的，但是我们的服务中大多数的应用场景都面临着读多写少的情况（二八法则）。Redis是将数据存储在内存中，所以读写速度非常快，若所有的业务数据直接与数据库相关，则会给数据库带来巨大的压力，而Redis正是能够缓解数据库的压力而存在，我们可以将一些常用的，变化不大的数据存入Redis中，在需要时直接从Redis中读取，提高我们系统的效率与稳定性，并且Redis支持多种数据类型，有着非常好的灵活性。在使用过程中，Redis提供了可选的数据持久化，能够在系统崩溃后，快速重新加载缓存，还提供了主从复制功能，可以实现高可用的分布式Cache系统。

Redis常用数据类型

在Redis中是适用一个redisObject对象来表示所有的key与valuew的，其中最主要的就是type与value对象，type代表着数据类型，value表示要存储的value。

  1. key:
    key的值是二进制安全的，所以任何二进制序列都可以作为key的值。所以key适合存储一些短小的数据，比如我们第三方应用的一些token、一些常用配置的数据等。
  2. String:
    String类型是二进制安全的，可以包含任何数据，如图片或者是序列化对象，也可以是数字。
  3. Hash:
    Redis支持hash方式存储数据，我们可以使用该特性来读取一些用户数据，如用户Id为key，value为用户的姓名、年龄、性别等信息并且计算hash后存入。
  4. List:
    Redis中的list类型的每一个元素都是String类型的双向链表，使用push与pop命令来添加与获取数据链表的头尾数据。使用List可以实现消息列表、消息排行等功能。
  5. Set:
    set是String类型的无需集合，可以最大存放2^32-1个元素，其原理是通过hash table实现，所以增删改成都是时间复杂度O(1)的操作，hash table也会随着set中的数据数量而变化。set还提供了求交集、并集、差集等操作，非常方便实用。
  6. SortedSet：
    与set相同，但是是具有排序功能。可以使用它来获取某些特定条件下的用户数据，如考试分数前50的学生、最受欢迎的用户等。

Jedis作用

使用jedis操控Pipeline

Redis本身是基于C/S模型的TCP通信方式，所以在client建立连接后可以发送多个请求至server端，但是每个请求都是阻塞的，所以client端会一直阻塞等待服务器返回，这样会非常低效，因为redis本质是高效的，但是每次阻塞并等待server端返回会将无用的处理时间用于在网络传输上，若网络延迟较大会大大的降低redis的效率。所以Pipeline出现了，使用Pipeline可使client一次性传输多条指令，server在收到一组请求后会存入消息队列中，执行完后一次性发送给client，其实就是异步交互方式，这样能够减少网络传输的开销，但是消息队列是存放在内存中的，如果使用过于频繁，容易GG。

jedis开启事务

reids的事务处理机制还是比较粗颗粒的，开启redis事务后，clinet向server发送的命令并不会立马执行，而是存放至server的队列中，当client调用exec命令时，队列中的命令会逐条执行。若执行失败则会忽略，其实就是序列化的执行命令，若中间有命令出错，就忽略该指令，继续执行下一条指令，不出错就继续执行，最后将结果打包返回给client。

redis中还有watch指令，类似于cas操作，在操作前添加watch指令，则在事务操作中，若数据发生了变化，会导致事务执行不成功。

jedis常用方法

string

  • set(key, value):存入一个数据
  • get(key):获取key的值
  • mget(key1, key2,key3…keyN)：返回多个key的值
  • setex(key, ex, value):设置一个有效时间为ex的key
  • mset(key1,key2..keyN,value1,value2,..valueN):批量设置多个key的值
  • incr(key):给key的string+1
  • decr(key): 为key的stirng-1
  • append(key): 在key后面增加value

    list

  • rpush(key, value): 在名称为key的list队尾增加一个value值
  • lpush(key, value): 在名称为key的lust队头增加一个value值
  • llen(key): 返回名称为key的list长度
  • larnge(key, start， end): 返回名称为key的list中下标为start-end的值
  • lindex(key, index): 返回名称为key的list中下表为index的值
  • lpop(key): 返回并删除名称为key的list中队尾的数据

    set

  • sadd(key, value): 向名称为key的set添加value值
  • srem(key, value): 删除名称为key的set中的value值
  • smembers(key): 返回名称为key的set中的所有元素
  • scard(key): 返回名称为key的set中值的数目
  • sinter(key1, key2):求名称为key1与key2的两个set的交集
  • sunion(key1, key2):求名称为key1与key2的两个set的并集
  • sdiff(key1, key2):求名称为key1与key2的两个set的差集

    hash

  • haset(key, value): 添加名称为key的hash的值
  • hget(key, value): 返回名称为key的hash的值
  • hexists(key, value): 查看名称为key的hash值是否有值

Redis常用场景

  1. 显示最新的消息列表
    比如某些文章、朋友圈状态，有最新回复后，就添加入redis的list中，当用户查看后就把该list清空。

  2. 排行榜
    比如微信运动的排行榜，可以适用redis的sorted set来进行排序去重，每个用户的好友都是不同的，所以每个用户都有自身的微信运动排行榜。

  3. 计数
    redis中有原子计数器，在进行增减时都是原子操作，所以不定担心会有并发问题。

  4. 常用用户信息存储
    有些用户信息是不常更新的,所以我们可以根据用户信息制定出合适的key，然后将该信息存入redis的hashMap中，在查询的时候可以直接查询Redis，而不用去关联一堆表到mysql中查询。
