写在开头

之前写过一篇文章介绍线程池，但是只是仅限于介绍，并没有深入的了解，所以这次深入源码分析JAVA线程池，在分析之前先把线程池的代码结构再梳理一遍。

Executor

Executor是线程池的顶级接口，只声明了execute方法，该方法传入一个runnable对象，用于执行线程池中的任务。

ExecutorService

ExecutorService继承了Executor接口，并在其基础上又声明了对线程池的各种操作，如submit、invokeAll、shutDown等方法。

AbstractExecutorService

AbstractExecutorService抽象类实现了ExecutorService接口中的声明方法。

ThreadPoolExecutor

ThreadPoolExecutor继承了AbstractExecutorService，它完成了对线程池的实现，是最UI最核心的一个类。

Executors

Executors是对ThreadPoolExecutor的再次封装，主要提供了各种类型线程池的工厂方法，方便我们调用。

ThreadPoolExecutor源码解析

线程池状态

ThreadPoolExecutor使用runState来标示当前线程池的状态，各个状态使用static final来声明：

  • RUNNING:运行状态，能够接收新的任务与执行队列中的任务
  • SHUTDOWN:关闭状态，不再接收新的任务但是继续执行队列中的剩余任务
  • STOP:停止状态，不接收与执行任务，若队列中还有等待执行的任务则会中断
  • TIDYING: 等待进入TERMINATED状态，TIDYING时任务队列为空，等待调用TERMINATED的hook函数
  • TERMINATED: 终止状态。线程池完全关闭

ThreadPoolExecutor中各个状态的转换时通过调用不同方法来完成的：

  • RUNNING -> SHUTDOWN 使用shutdown()方法
  • (RUNNING or SHUTDOWN) -> STOP 使用shutdownNow()方法
  • SHUTDOWN -> TIDYING 当工作队列或线程池没有任务与线程的情况下
  • STOP -> TIDYING 当线程池没有线程工作
  • TIDYING -> TERMINATED 当使用terminated()钩子方法来完成

ThreadPoolExecutor中的属性

ThreadPoolExecutor中还定义了许多属性，用于在构建线程池时定义任务的缓存策略、线程存活时间、任务执行策略等

  • BlockngQueue workQueue：任务缓存队列，用来存放等待执行的任务
  • ReentrantLock mainLock: 线程池的主要锁状态。
  • HashSet workers：用来存放工作集，线程池里可以立即执行的任务放在这个集合里，它本身实现了runnable接口，可以把它当成一个线程来使用。
  • keepAliveTime: 线程存活时间，超出该时间可以执行一些抛弃策略
  • boolean allowCoreThreadTimeOut: 是否允许为核心线程设置存活时间
  • int corePoolSize: 核心core大小
  • int maximumPoolSize: 线程池能够最大容忍的线程数
  • int poolSize: 线程池当前的线程数
  • RejectedExecutionHandler handler:线程池的任务拒绝策略
  • ThreadFactory threadFactory:线程工厂，用来创建线程
  • int largestPoolSize:记录线程池中曾经出现过的最大线程数
  • long completedTaskCount:用来记录已经执行完毕的任务个数

关闭线程池

shutdown与shutdownnow都是关闭线程池的方法，但是它们在操作上很不同。
shutdown方法用于关闭线程池，调用shutdown方法后，线程池将不再接收新的任务，但是在任务队列中的任务还是会继续执行，直到全部执行完毕

shutdownNow方法是立即关闭线程池，并不会等待任务执行完毕，而是将任务队列中的任务全部删除，并尝试关闭线程池。
