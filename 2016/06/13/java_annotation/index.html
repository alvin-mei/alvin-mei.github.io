发表于 2016-06-13   |   分类于 Annotation ， java   |     |   阅读次数

什么是注解？

注解就是一组元数据，用于描述数据本身的数据，在不少开源库中都使用到了注解的方法来简化代码，如Spring、SpringMVC等，通过注解的方式能够使代码更加的清晰，并且能够提高代码开发效率。

注解的应用

  1. 作为特定的标记：告诉编译器一些信息
  2. 额外的信息的载体：比如override，可以供编译器使用，提供关于代码更多的检查与验证
  3. 运行时动态处理：得到注解的信息

注解分类

  1. 标准注解:Java自带的几个Annotation，用于标识某些类的信息，所以没有参数输入。如override，标识该方法需要覆盖父类的同名方法。
  2. 元注解: 可以设置参数，提供更为强大的控制功能，一般用于标注代码、标注注解。
  3. 自定义注解: 根据需求，自定义注解
    • 文档标注
    • 编译检查
    • 代码分析

元注解详解

元注解是用来负责注解其他注解，也就是描述注解的注解，很绕口吧，Java5.0定义了4个标准的元注解类型，具体分类如下：

  1. @Target
  2. @Retention
  3. @Documented
  4. @Inherited

Target

这个元注解用来描述注解锁修饰对象的范围，表明注解目标是什么类型。他的参数是一个枚举类型ElementType，具体的值如下:

  • TYPE: 类
  • FIELD: 字段（可以是对象）
  • METHOD: 方法
  • PARAMETER: 参数
  • CONSTRUCTOR: 构造方法
  • ANNOTATION_TYPE: 注解
  • PACKAGE: 包

Retention

Retention用来定义Annotation的生命周期，也可以理解为该注解在什么状态下运行。它的参数是一个枚举类型RetentionPolicy,具体值如下：

  • SOURCE: 源码文件中有效
  • CLASS: class文件中有效
  • RUNTIME: 运行时有效

一般来说，Retention的SOURCE与CLASS注解和编译器行为有关，比如报错、警告、类编译参数等，而RUNTIME一般开发人员使用的比较多，比如在RUNTIME条件下，那么就可以使用反射获取类信息等。

Documented

描述Annotation时说明该注解是可以被文档化的

Inherited

用于表明该注解是可以被继承的，如一个使用了被@Inherited标注的注解的类，那么它的子类也将获得该注解的功能。
