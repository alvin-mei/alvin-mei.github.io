单例模式的思考

发表于 2016-06-16   |   分类于 设计模式 ， java   |     |   阅读次数

单例模式有什么好处？

单例模式的好处呢就是在整个程序的生命周期中只存在一份实例，也就是说我们希望一个方法、对象与其所在类的实例对象无关。所以呢该类能够保证数据的正确性而且能够节省内存资源，比如系统在初始化时就需要加载的一些配置与属性，在整个程序生命周期中这些配置与属性都是公共的，且在程序运行期间都一直存在一份拷贝。

如何实现一个单例模式？

其实单例模式的实现方式非常的简单，我们只需要将其设置为静态的对象、方法就可以了，然后只需要提供一个对外的方法来获取该实例，主要步骤有以下三点：

  • 私有化静态对象
  • 私有化其构造函数:防止被外部方法调用，生成实例。
  • 公有化一个静态的工厂方法，用于获取单例对象。

单例模式实现方式

单例模式有许多中实现方式，每一种实现方式都有其各自的特点，下面让我们来看以下如何正确实现单例模式

懒汉式

懒汉式单例模式呢，我们的实例在一开始时是没有生成的，而是当被调用的时候才会生成，这种延迟加载特性可以用在配置文件之类的功能上。但是这里会有并发问题，当多个线程都调用getInstance时，就会生成多个实例。

饿汉式

饿汉式单例模式呢就如它的名字一样，事先就将对象准备好，来一个调用就直接将instance给对方。饿汉式本身是线程安全的，但是一开始就要加载，会浪费许多空间。

双重检验锁

使用双重检验锁来保证单例模式的线程安全，虽然说效率低下了一点，但是毕竟靠谱啊！

静态内部类方式

由于双重检验锁方式的开销过大，所以可以使用静态内部类的方式来代替它。同样是线程安全，在系统加载Singleton时，我们所需的Instance并不会被初始化，因为利用了静态内部类的特性，只有在SingletonHolder被主动调用时，才会生成实例并返回。

总结

单例模式是用来实现在程序生命周期中只有一个实例的设计方法。

  • 单例模式的要点有三个：一是某个类只能有一个实例；二是它必须自行创建这个实例；三是它必须自行向整个系统提供这个实例。单例模式是一种对象创建型模式。
  • 单例模式适用情况包括：系统只需要一个实例对象；客户调用类的单个实例只允许使用一个公共访问点。
