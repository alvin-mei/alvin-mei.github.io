一、Condition简介

Condition也就是信号量，每一个Lock对象都可以拥有Condition对象，通过Condition可以使线程之间进行通信，可以说Condition是一个线程间通信的工具类。它可以使线程在某些条件下进入条件队列，从而变成阻塞状态，而又在达到一定的条件后重新被唤醒，从条件队列中退出并重新夺取锁，从而拥有执行权。通过Condition我们可以对线程的执行顺序，执行情况进行精准控制。我们先通过下面的例子来了解下Condition的功能

三、AQS对Condition的实现

AQS中的ConditionObject方法实现了Condition接口，在ConditionObject中定义了两个CLH队列中的节点,在AQS中使用CLH队列来管理Condition条件顺序

下面看下AQS如何实现Condition接口的方法

await

从上述代码我们可以看到，整体流程如下：

  1. 判断当前线程是否中断，若中断则抛出异常
  2. 构造Condition队列中的节点，并将当前线程所占用的锁释放
  3. 这时当前线程已经进入Condition队列，等待signal通知唤醒，所以会一直检查该线程是否重新进入AQS队列，若不是则调用park方法进行阻塞，在此过程中，若发生中断则会跳出程序。
  4. 若当前线程已经获取通知并唤醒，那么就会去重新尝试获取锁，若获取到了那么就会清理Condition队列中的无用队列，并检查interruptMode的状态是否有问题，没有问题则安全退出。

signal方法

在Condition中使用signal方法来将Condition队列中的第一个节点唤醒，下面看一下代码，依旧由AQS来实现

可以看到signal方法主要流程如下：

  1. 获取Condition队列中第一个节点
  2. 利用获取到的第一个节点，遍历Condition队列，找到第一个能够唤醒的节点
  3. 找到能够唤醒的节点后将其加入到AQS队列中，并唤醒使他去尝试获取锁

而signalAll方法则是将Condition中所有可以唤醒的节点都加入到AQS队列中，原理与signal差不多，这里就不详细介绍了。
通过查看源码可以发现，Condition其实就是维护了一个等待响应的队列，将调用await的方法的线程都加入到该队列中并解除所持有的锁并阻塞，然后等待其他线程调用signal来将其唤醒，若唤醒则从Condition队列中移除，并加入AQS队列中去竞争锁。
